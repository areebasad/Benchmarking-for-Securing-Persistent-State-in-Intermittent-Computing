1. Start from examples specifically from Atmel Start
2. Add modules as per requiremnet
3. If using USART module look for device receiving and transmiting pins. (Select the right one, see USART component in datasheet for help)
For Cortex M0+
RX - PA23
TX - PA22

4. Links on Microchip website are not updated (See documnets year)
ASF4 API OR ASF Manual (SAM L21)

5. For measuremnets include gpio_init.c/h files
Change DGI GPIO port pins according to PINMUX (Get help from datasheet or atmel start)
include the header of gpio_init.c in atmel_start.h file and
define "#define INIT_GPIO" in atmel_start_pins.h file


6. For WolfSSL Integration

1. For just AES you should only need the following source files from wolfssl-4.3.0/wolfcrypt/src/
aes.c
error.c
hash.c
logging.c
memory.c
misc.c
wc_encrypt.c
wc_port.c
  

Then use these settings to configure the library for AES only by making a file named "user_settings.h" and placing the below defines in that file. 
Then globally define WOLFSSL_USER_SETTINGS (or add a line #define WOLFSSL_USER_SETTINGS at the top of wolfssl-4.3.0/wolfssl/wolfcrypt/settings.h) and make sure to #include <wolfssl/wolfcrypt/settings.h> in your application
before all other wolfSSL headers.
